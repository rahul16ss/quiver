# Acceptance Criteria: Compliance Review

Copy this file to `.quiver/acceptance.md` in your workspace to define what a
complete, high-quality compliance review looks like. The maker-checker will
verify each criterion before the agent writes the final deliverable.

## Structure
- [ ] Review has a clear title, scope, and date
- [ ] Review identifies the regulatory framework(s) being assessed against
- [ ] Review has an executive summary of findings
- [ ] Review has a detailed findings section with each issue numbered
- [ ] Review has a recommendations section with prioritized actions
- [ ] Review has an appendix listing all sources consulted

## Findings
- [ ] Each finding has a unique identifier (e.g., FINDING-001)
- [ ] Each finding cites the specific regulation or standard it relates to
- [ ] Each finding has a severity rating (Critical / High / Medium / Low)
- [ ] Each finding describes the gap between current state and required state
- [ ] Findings are based on evidence, not speculation

## Recommendations
- [ ] Each recommendation maps to a specific finding
- [ ] Recommendations are actionable (not "be more careful")
- [ ] Recommendations have a suggested owner and timeline where possible
- [ ] Recommendations are prioritized by risk severity

## Sources
- [ ] Every regulatory citation includes the specific section/clause
- [ ] Sources include the actual regulation text, not secondary commentary
- [ ] At least 3 distinct regulatory sources are cited
- [ ] All source URLs are valid and accessible

## Format
- [ ] Review is written in formal compliance language
- [ ] No placeholder text (TODO, FIXME, lorem ipsum)
- [ ] No unexplained acronyms
- [ ] Review length is appropriate (minimum 1500 words for substantive reviews)
