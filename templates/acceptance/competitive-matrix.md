# Acceptance Criteria: Competitive Matrix

Copy this file to `.quiver/acceptance.md` in your workspace to define what a
complete, high-quality competitive matrix looks like. The maker-checker will
verify each criterion before the agent writes the final deliverable.

## Structure
- [ ] Matrix has a clear title, market definition, and date
- [ ] Matrix identifies at least 5 competitors
- [ ] Matrix has at least 5 comparison dimensions
- [ ] Matrix has a summary analysis section
- [ ] Matrix has a methodology note explaining how data was collected

## Comparison Dimensions
- [ ] Pricing is compared (with source citations)
- [ ] Key features are compared (with specific details, not vague claims)
- [ ] Target market / customer segment is identified for each competitor
- [ ] Funding or revenue scale is noted where available
- [ ] Geographic coverage is noted

## Data Quality
- [ ] Every data point has an inline citation with source URL
- [ ] Data is current (within 12 months unless historical context needed)
- [ ] Sources are primary where possible (company websites, filings, press releases)
- [ ] Conflicting data from different sources is acknowledged
- [ ] Data gaps are explicitly noted (not silently omitted)

## Analysis
- [ ] Matrix includes a positioning analysis (where each competitor sits)
- [ ] Matrix identifies competitive advantages and disadvantages
- [ ] Matrix notes market trends affecting the competitive landscape
- [ ] Recommendations or observations are clearly separated from data

## Format
- [ ] Matrix is presented as a structured table
- [ ] No placeholder text (TODO, FIXME, lorem ipsum)
- [ ] Professional tone, no marketing language
- [ ] Minimum 1000 words including analysis
