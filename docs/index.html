Quiver

An open foundation for controlled, source-backed document workflows in finance.

The problem

Generic chat tools fall short for investment teams: figures appear without a trail, the model does not know the firm's template or prior work, and the output stays in a chat window instead of the Word, Excel, and PowerPoint files a senior professional actually reviews and signs. The limitation is not drafting speed — it is evidence, format, and control.

What Quiver is

Quiver helps turn approved files, public research, workflow instructions, and review rules into native Office deliverables with inspectable evidence. Conviction Studio uses it as one foundation for bespoke client workflow engagements.

Quiver is infrastructure, not investment advice and not a turnkey replacement for professional judgment. Generated work remains a draft until a professional reviews it.

Flagship example

The repository ships one end-to-end example — an investment committee memo for a synthetic company. From a template, an Excel model, a filing excerpt, a transcript, and a vendor-style export, it generates a native Word memo, an evidence map connecting figures to sources (Excel-sourced figures verified against the actual model cells), a review checklist of flagged and unresolved items, and a run record — then verifies the output with acceptance checks.

npm run demo:ic-memo — no API keys or network required.

Principles

  • Controlled context — understand and control the files, instructions, sources, and memory a workflow uses.
  • Inspectable evidence — important figures and claims connect to sources or are explicitly flagged for review.
  • Native deliverables — Word, Excel, and PowerPoint, not only chat.
  • Human responsibility — output is a draft until a professional reviews it.
  • Data boundaries — model use, local processing, remote calls, and approved sources are configured around the workflow.
  • Model independence — workflow context and process survive a provider change.

Data handling

By default the primary model is served from a cloud endpoint, so prompt content is sent to that provider. Memory, sessions, documents, and the tamper-evident audit log live in files on your machine. Local model endpoints are supported where an engagement requires them. There is no telemetry. Data handling and model use are configured around the workflow's sensitivity.

Install

git clone <repository> && cd quiver
npm install
npm run demo:ic-memo   # flagship workflow + acceptance checks
npm run gui            # desktop app

Prerequisites: Node 20+ and OfficeCLI for native Office output.
