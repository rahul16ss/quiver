Story 02

Zero-Friction Lens Swapping

Intelligence is a layer, not a lock-in. Switch model providers, base URLs, or local endpoints on the fly. The harness maintains the session context and memory files seamlessly across compute layers.

User Story Details

As a researcher running expensive runs,

I want to toggle active endpoints instantly via the `/model` CLI command,

So that I can offload complex reasoning to cloud gates and swap back to local models for quick edits.

  • Real-time status transitions.
  • Seamless continuation of session memories.

Open vs. Closed Harnesses

Stop letting closed SaaS layers gatekeep your agent's memory and execution steps.

Proprietary Wrappers

  • Trapped in a single closed provider ecosystem.
  • Prompt context and agent histories locked in private databases.
  • System prompts are assembly lines hidden from developers.
  • Custom tools require proprietary JSON platform APIs.

Quiver open harness

  • Model sovereignty: swap local or cloud providers instantly.
  • Memory is local Markdown files under your version control.
  • Prompt structure fully detailed before execution.
  • Dynamic tools are simple TypeScript files.

Harness Principles

We align Quiver's architecture with clean, code-enforced rules of control and explainability.

01

Local Context Control

Your agent's memory, persona, and logs live in directories on your drive, sync naturally through your local folders, and belong completely to you.

02

Context Transparency

No secret prompts. The resolved context bundle entering the model call is logged openly before every turn.

03

Total Explainability

Chronological trace logs record exactly what prompts caused tool actions and the stdout returned by the OS.

04

Low-Level Primitives

When tool capabilities are missing, the agent doesn't compromise—it builds the TypeScript code wrapper it needs at runtime.

05

Maker-Checker Sandbox

High-risk code changes are executed in isolated copy-on-write scratchpads, validated by checker subagents, and subject to automatic rollback on failures.

06

Hardened Credentials

API keys are isolated from child processes, redacted in session logs, and securely resolved from macOS, Linux, or Windows keychains.

Harness Commands

Take active control of your sessions with built-in commands. Learn More →

Command Description
/help Show helper manual detailing command inputs.
/tools Display list of all compiled tool wrappers in the registry.
/config Prints configurations, active models, and paths.
/model <name> Swaps the active LLM choice (e.g. /model llama3).
/compact Runs text compaction on intermediate trace logs.
/reset Clears execution context without deleting memory folders.
/resume Prompts a selection of saved runs to resume.
/exit Closes the session stream, dumps trace files, and exits.

Runtime Flags

Configure execution behavior when launching Quiver sessions.

Flag Description
--continue, -c Immediately restarts the most recently active session log.
--resume <id>, -r <id> Resumes a specific session by its file ID directly.
--list-sessions Lists all local JSON session logs with token stats.
--single-turn "prompt" Executes the input instruction directly in the terminal and exits.
--json Formats terminal responses to structured JSON for scripting.
--dry-run, -n Simulates tool choices and shows arguments without modifying any files.
