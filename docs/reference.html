Architecture

Five Primary Systems

1. Filesystem Context Manager

src/context_manager.ts & src/paths.ts — All project-specific data is sandbox-isolated. Memory is organized as plain text files inside ~/.quiver/projects/{project_id}/memory/.

2. Harness-Adapter & Provider Split

src/adapters/ & src/providers/ — Decouples transport (Provider) from alignment (Adapter). Provider handles streaming, auth, token counting. Adapter handles prompt shapes, tool formats, citation styling.

3. Lifecycle Hooks Interception Engine

src/lifecycle.ts — Unified middleware/hook system: beforeAgent, beforeModel, wrapModelCall, wrapToolCall, afterModel, afterAgent.

4. Active Timeouts & Guardrails

src/agent.ts — Model-specific timeouts, stream stall protection, hash-based read-before-write verification.

5. Dynamic Tool Sandbox Registry

src/registry.ts — Single-purpose TypeScript tools. Out-of-process execution, JIT compilation via esbuild (<200ms), leak-free hot-reloading.

Security

Security Boundaries

Path Sandbox

Canonicalizes paths, resolves symlinks, verifies target inside workspace root. Blocks .env, .git/, ~/.ssh/, credentials by default.

Command Risk Classification

Shell commands classified: safe, moderate, network, destructive, privileged, secret-risk, exfiltration-risk. High-risk always requires approval.

Secret Redaction

Detects AWS keys, SSH keys, API keys, Bearer tokens. Redacts before logging, syncing, or remote transmission.

Prompt Injection Defense

File contents wrapped in <untrusted_file> tags. System prompt specifies untrusted content cannot override harness instructions.

Atomic Writes with Rollback

Temp-write-then-rename pattern. Backup created before overwrite. /rollback last restores previous state.

Memory

Memory Architecture

Core Memory

~/.quiver/core.json — Global identity and human context.

Project Memory

~/.quiver/projects/{project_id}/memory/ — Per-project memory files: persona.txt, user-preferences.md, workspace-facts.md, facts.jsonl.

Session Memory

~/.quiver/projects/{project_id}/.sessions/ — Session logs, checkpoints, compacted archives.

Privacy Labels

public — Safe for remote & sync | project — Project-scoped | private — Opt-in remote | secret — Never remote, never synced

Configuration

Configuration

Environment Variables

A single OLLAMA_API_KEY powers the LLM, Ollama, and vision adapters. LLM_API_KEY, VISION_MODEL_API_KEY, and CONTEXT7_API_KEY are retired.

OLLAMA_API_KEY — API key (required; stored in keychain when available)

LLM_API_BASE_URL — Model provider endpoint

LLM_MODEL_NAME — Model name (source-controlled default; override only)

VISION_MODEL_NAME — Vision model (source-controlled default; override only)

VISION_MODEL_BASE_URL — Vision endpoint

QUIVER_AUTONOMY — Comma-separated autonomy grants (write_file, run_command, destructive, etc.) or a developer trust tier (see docs/advanced.md)

QUIVER_MAX_CONTEXT_TOKENS — Context window limit

QUIVER_AUTONOMY=browser:visible — Show the browser window (otherwise headless)

QUIVER_SESSION_LOG / QUIVER_SESSION_LOG_MAX_CHARS — Session logging

PARALLEL_API_KEY — Optional; powers web search, scrape, deep research

GITHUB_TOKEN — Optional; GitHub tooling (developers only)
